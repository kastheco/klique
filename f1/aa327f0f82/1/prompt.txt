## Plan Context

**Goal:** Include review cycle numbers in instance titles, tmux session names, and opencode session labels (e.g. `-review-1`, `-fix-1`, `-review-2`) so each review/fix cycle gets unique identifiers, preventing tmux session clashes, stale caches, and opencode DB collisions.
**Architecture:** Add a `ReviewCycle` counter to `PlanEntry` (persisted via `review_cycle` column in planstore SQLite). The counter starts at 0 and increments on each `ReviewChangesRequested` FSM transition. The three spawning functions (`spawnReviewer`, `spawnCoderWithFeedback`, `spawnPlanAgent` review case) read the cycle count and append it to instance titles. `BuildTitle` in `opencodesession` gains an optional `ReviewCycle` field for human-readable labels like `review #2`. The first review cycle (cycle 0) uses suffix `-review-1` / displays `review #1` (1-indexed for humans).
**Tech Stack:** Go, SQLite (planstore), bubbletea TUI, tmux session management
