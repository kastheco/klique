## Plan Context

**Goal:** Fix the history toggle in the sidebar navigation panel so that pressing Right on an expanded history section descends into the first child row, matching the behavior of plan headers and the dead toggle.
**Architecture:** The `Right()` method in `ui/navigation_panel.go` handles arrow-key expansion/descent for collapsible rows. The `navRowHistoryToggle` case is missing the `else { n.Down() }` branch that `navRowPlanHeader` and `navRowDeadToggle` both have. Adding the missing branch and a test covers the fix.

**Step 1: write the failing test**

Add this test near the existing navigation tests in `ui/nav_panel_test.go` (close to `TestNavigation_RightExpandsAndDescends`) and reuse existing helpers `newTestPanel()` and `SetData(...)`:

```go
func TestRight_HistoryToggle_ExpandedDescendsIntoChild(t *testing.T) {
	n := newTestPanel()
	history := []PlanDisplay{
		{Filename: "old-a.md"},
		{Filename: "old-b.md"},
	}
	n.SetData(nil, nil, history, nil, nil)

	// Expand history section first.
	n.selectedIdx = 0
	n.ToggleSelectedExpand()
	require.True(t, n.historyExpanded)
	require.Len(t, n.rows, 3) // toggle + 2 history plans

	// Select the toggle row again and press Right — should descend.
	n.selectedIdx = 0
	n.Right()
	assert.Equal(t, 1, n.selectedIdx, "Right on expanded history toggle should move to first child")
	assert.Equal(t, navRowHistoryPlan, n.rows[n.selectedIdx].Kind)
}
```

Notes for imports: no new imports are needed beyond what `ui/nav_panel_test.go` already uses (`testing`, `session`, `assert`, `require`).
