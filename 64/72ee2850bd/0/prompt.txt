Implement Task 1: Make waveAllCompleteMsg push non-blocking

Load the `kasmos-coder` skill before starting. Also load `cli-tools` for the tool selection reference.

## Plan Context

**Goal:** Eliminate the 1-2s UI freeze that occurs when the user confirms the "push branch and start review?" dialog after wave completion, caused by a synchronous `git push` in the bubbletea `Update` loop.
**Architecture:** The `waveAllCompleteMsg` handler in `app/app.go` calls `worktree.Push(false)` synchronously inside `Update()`, blocking the TUI render loop for the duration of the network round-trip. The fix moves the push into an async `tea.Cmd` that returns a new `wavePushCompleteMsg` on completion, then the FSM transition and reviewer spawn happen in that message's handler. This mirrors the existing pattern used by `coderCompleteMsg` / `promptPushBranchThenAdvance` in `app/app_state.go`.
**Tech Stack:** Go, bubbletea (tea.Cmd async pattern), git subprocess

2. Rewrite `case waveAllCompleteMsg:` so it does **only** async push kickoff and returns a command. Do not transition FSM, refresh sidebar, or spawn reviewer in this case.

Use this behavior contract:
- Resolve the target plan file once (`planFile := msg.planFile`).
- Find one matching task instance (`inst.TaskFile == planFile && inst.TaskNumber > 0`) to obtain shared worktree context.
- In the returned `tea.Cmd`, do best-effort push (`GetGitWorktree`, `Push(false)`), swallowing errors exactly like existing code paths.
- Always return `wavePushCompleteMsg{planFile: planFile}` even if no instance/worktree/push is available.

Concrete implementation shape:

```go
case waveAllCompleteMsg:
    planFile := msg.planFile

    var pushInst *session.Instance
    for _, inst := range m.nav.GetInstances() {
        if inst.TaskFile == planFile && inst.TaskNumber > 0 {
            pushInst = inst
            break
        }
    }

    return m, func() tea.Msg {
        if pushInst != nil {
            if worktree, err := pushInst.GetGitWorktree(); err == nil && worktree != nil {
                _ = worktree.Push(false)
            }
        }
        return wavePushCompleteMsg{planFile: planFile}
    }
```
