**Context:** The planner skill template (source of truth for all synced copies in `.agents/` and `.opencode/`) currently contains three problems related to import dependencies:

1. **Line 226** — `the "dependency" is just imports (the compiler catches that)` in the "when NOT to use multiple waves" section. This actively tells planners to ignore import dependencies. But when Task A creates `type Foo` in `app/model.go` and Task B in `ui/render.go` imports it, they cannot run in parallel — Task B will fail to compile until Task A's code exists.
