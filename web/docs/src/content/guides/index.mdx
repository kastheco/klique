# guides

Practical walkthroughs for day-to-day kasmos usage. These guides assume you have already completed `kas setup` and can launch the TUI with `kas`. If you haven't done that yet, start with [getting started](/docs/getting-started).

## in this section

- [tui overview](./tui-overview) — keymap reference, layout, panes, and how to navigate the interface
- [creating plans](./creating-plans) — how to describe work, spawn a planner, and get a structured plan into the store
- [running implementations](./running-implementations) — normal wave-based flow, blueprint skip for small tasks, and the elaboration phase
- [reviewing and merging](./reviewing-and-merging) — reviewer spawn, approval vs changes requested, pull requests, and merge
- [managing instances](./managing-instances) — start, attach, detach, pause, resume, checkout, kill, stop, and adopt orphaned sessions

## related reference

- [concepts](/docs/concepts) — mental models for tasks, waves, topics, lifecycle, agents, and worktrees
- [cli reference](/docs/cli-reference) — full `kas` command documentation
