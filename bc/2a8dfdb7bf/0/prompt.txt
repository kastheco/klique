Implement Task 1: Remove date prefix from filename generation, DisplayName, Rename, and all tests

Load the `kasmos-coder` skill before starting. Also load `cli-tools` for the tool selection reference.

## Plan Context

**Goal:** Stop prepending `YYYY-MM-DD-` to plan filenames so that working on a plan across multiple days doesn't cause slug mismatches; the slug alone is a sufficient unique identifier.
**Architecture:** The date prefix is generated in 4 places: `buildPlanFilename()`, `createPlanEntry()`, `importClickUpTask()`, and `ScaffoldFilename()`. `DisplayName()` strips it for display, and `Rename()` preserves it. All of these must be updated to produce dateless slugs (e.g. `auth-refactor.md` instead of `2026-02-21-auth-refactor.md`). `DisplayName()` becomes a simple `.md` strip. `Rename()` drops the date-preservation logic. The planner skill template, planner agent prompt, and ~20 test files need their hardcoded `2026-XX-XX-` prefixed filenames updated. The `dedupePlanFilename` functions already work on the base slug so they need no logic changes, only test expectation updates.
**Tech Stack:** Go, bubbletea (TUI), planstate/planstore packages, kasmos-planner skill (Markdown)
